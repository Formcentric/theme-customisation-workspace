# Ein eigenes Theme erstellen

Erstellen Sie ein eigenes Theme mit folgendem Befehl:

```bash
pnpm create-theme
```

### Extended Formcentric-Theme

- Ideal für gezielte Style-Anpassungen an einem bestehenden Theme
- Erbt die meisten Styles vom gewählten Basis-Theme
- Ermöglicht das gezielte Überschreiben von Komponenten und Variablen
- Geringster Wartungsaufwand

### Full Formcentric-Theme

- Vollständige Kopie eines bestehenden Themes
- Volle Kontrolle über alle Theme-Komponenten
- Höherer Wartungsaufwand bei Änderungen am Basis-Theme
- Gute Balance zwischen Flexibilität und Komplexität

## Best Practices

1. **Modulare Struktur**

    - Teilen Sie Ihre Styles in logische Komponenten auf
    - Verwenden Sie Partials für bessere Wartbarkeit
    - Befolgen Sie eine einheitliche Namensgebung

2. **Nutzung von Variablen**

    - Setzen Sie Theme-Variablen für konsistentes Styling ein
    - Machen Sie Ihre Styles konfigurierbar über `definition.json` (Formcentric Cloud) und die Dateien `_fc-variables.scss`, `_variables.scss` und `_variables.json`
    - Verwenden Sie SCSS-Features, um DRY-Prinzipien umzusetzen

3. **Responsives Design**

    - Berücksichtigen Sie Breakpoints für Mobilgeräte und Desktop
    - Nutzen Sie flexible Einheiten wie em
    - Testen Sie auf verschiedenen Bildschirmgrößen

4. **Barrierefreiheit (Accessibility)**

    - Berücksichtigen Sie Fokus-Zustände
    - Stellen Sie ausreichende Farbkontraste sicher
    - Unterstützen Sie Tastaturnavigation bei Ihren benutzerdefinierten Templates
